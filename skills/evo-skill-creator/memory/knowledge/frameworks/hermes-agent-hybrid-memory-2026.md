## 三层混合记忆架构（OS-Memory 模型）

```
┌────────────────────────────────────────────┐
│         Core Memory（核心记忆）              │
│     = RAM：始终在上下文窗口中               │
│     内容：任务目标、系统指令、关键状态变量   │
│     容量：小（~10% context window）         │
├────────────────────────────────────────────┤
│      Recall Memory（回忆记忆）              │
│     = 磁盘缓存：可搜索的近期上下文         │
│     内容：对话历史、近期语义片段            │
│     容量：中等                              │
├────────────────────────────────────────────┤
│     Archival Memory（归档记忆）             │
│     = 冷存储：向量索引的长期知识           │
│     内容：归档信息、历史知识               │
│     容量：大                               │
└────────────────────────────────────────────┘
```

### 各层特征

| 层次 | OS 类比 | 访问速度 | 容量 | 更新频率 |
|------|---------|---------|------|---------|
| Core Memory | RAM | 即时（在上下文中） | 小 | 高 |
| Recall Memory | 磁盘缓存 | 快（语义搜索） | 中 | 中 |
| Archival Memory | 冷存储 | 慢（RAG 检索） | 大 | 低 |

1. 从"全量历史保留"���向**智能压缩**
2. **语义压缩**：将交互蒸馏为索引单元

4. **Token 预算管理**：在 Token 限制内最大化上下文深度

## 与我们的架构映射

| 三层架构 | 我们的实现 | 差距 |
|---------|-----------|------|
| Core Memory | SKILL.md + 唤醒时加载的核心 memory 文件 | 缺少动态调整能力 |
| Recall Memory | memory/ 目录下的各文件 | 缺少语义搜索，依赖 Grep |
| Archival Memory | 无 | **完全缺失** |

### 我们的优势

1. **文件系统天然分层**：SKILL.md 是 Core，memory/ 是 Recall，无需额外基础设施
2. **人类可读**：所有记忆都是 Markdown，可直接检查和修改
3. **零依赖**：不需要向量数据库等外部服务

### 我们的差距

1. **Core Memory 不够动态**：每次都加载固定文件，不根据任务调整
2. **Recall Memory 无语义搜索**：只能靠文件名和 Grep，无法理解语义
3. **Archival Memory 缺失**：过时知识没有归档机制
4. **缺少智能压缩**：知识文件只增不减，没有蒸馏和压缩

### 改进方向

1. **短期**：
   - 在唤醒逻辑中增加"任务相关性判断"——根据当前命令和参数决定加载哪些 memory 文件
   - 为 knowledge/ 增加 INDEX.md 索引文件

2. **中期**：
   - 实现知识压缩机制（合并相似知识、摘要长文件）
   - 引入归档目录 `memory/archive/`，存放过时但不删除的知识
   - 基于 Grep 实现简单的语义检索工具

3. **长期**：
   - 评估是否需要轻量级向量索引（本地 Embedding）
   - 实现动态 Core Memory 调整
