# SAGE 框架与 Memory 管理机制

> 学习日期：2026-03-26
> 来源：SAGE 论文（2025年4月）+ Memory 管理领域综合研究

## SAGE 框架核心

### 架构：三 Agent 协作

- **User Agent**：发起任务
- **Assistant Agent**：执行任务
- **Checker Agent**：验证和反馈

三者通过 self-evolution 闭环协作，无需额外训练即可持续改进。

### 艾宾浩斯遗忘曲线机制

SAGE 将遗忘曲线应用于 memory 管理：
- **新信息衰减快**：刚获取的信息如果不被使用，权重迅速降低
- **反复使用的信息留存**：被多次调用的知识权重升高
- **定期温故**：根据遗忘曲线计算最佳温故时间点

### 反思机制

- Agent 在执行任务后自我反思：做对了什么？做错了什么？
- 反思结果用于优化后续决策
- 减少"灾难性遗忘"（在学习新知识时丢失旧知识）

### 成果

- 闭源模型性能提升 2.26 倍
- 开源模型提升 57.7%~100%
- 对小模型效果尤为显著

## Memory 管理前沿研究

### 核心 Memory 操作（不只是存储）

| 操作 | 说明 |
|------|------|
| 存储 (Store) | 写入新知识 |
| 检索 (Retrieve) | 按需取出相关知识 |
| 整合 (Consolidate) | 将零散信息合并为结构化知识 |
| 更新 (Update) | 修正已有知识 |
| 遗忘 (Forget) | 主动丢弃过时/不相关信息 |
| 压缩 (Condense) | 将详细信息精炼为摘要 |

### 关键框架

#### Focus（主动上下文压缩）
- Agent 自主决定何时将学习成果整合到"Knowledge"块
- 主动裁剪原始交互历史
- 灵感来源：生物导航模式

#### AgeMem（统一长短期 Memory）
- 将 memory 操作暴露为工具级动作（store/retrieve/update/summarize/discard）
- Agent 自主决定存什么、何时存、何时丢弃
- 复杂多步场景中表现显著提升

#### Memory 进化三阶段框架
1. **Storage**（轨迹保存）：原始记录所有交互
2. **Reflection**（轨迹精炼）：反思后提取关键信息
3. **Experience**（轨迹抽象）：抽象为可复用的经验规则

## 对 evo-skill-creator 的启示

### Memory 管理策略设计（直接服务于 backlog #3）

应该实现的 memory 操作：

```
learn/scan 执行后：
  → 新知识写入 knowledge/（Storage）
  → 标记"首次获取时间"和"最近访问时间"

go 执行前读取 memory ��：
  → 更新"最近访问时间"（相当于温故）
  → 长期未访问的知识标记为"冷数据"

review 执行时：
  → 检查冷数据 → 决定压缩/归档/删除
  → 将多个零散知识整合为结构化总结
  → 反思：哪些知识真正用到了？哪些从未被引用？
```

### 具体建议

1. **给每个知识文件加元数据**：创建时间、最近访问时间、被引用次数
2. **review 命令增加 memory 健康检查**：识别从未被引用的知识、过长的文件、重复内容
3. **设计"压缩"操作**：将细节丰富但很少被读的文件精炼为摘要版
4. **参考 SAGE 的"先温故再学新"原则**——我们已经在 learn 命令中实现了这一点

### 我们目前的差距

| 操作 | 我们是否支持 | 说明 |
|------|-------------|------|
| 存储 | 是 | learn/scan 写入 knowledge/ |
| 检索 | 部分 | 唤醒时读 memory，但不够精准 |
| 整合 | 否 | 没有合并/结构化机制 |
| 更新 | 部分 | learn 先温故再增量 |
| 遗忘 | 否 | memory 只增不减 |
| 压缩 | 否 | 没有精炼/摘要机制 |
