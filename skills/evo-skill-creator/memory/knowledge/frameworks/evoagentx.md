# EvoAgentX 框架——自动化评估与进化循环

> 学习日期：2026-03-26
> 来源：EvoAgentX GitHub + 论文（ACL Anthology / arXiv）

## 核心概念

EvoAgentX 是一个开源框架，用于**自动化构建、评估和进化** LLM agent 及多 agent 工作流。核心理念：从静态模型到自我进化生态系统。

## 五层模块化架构

```
Layer 5: Evaluation     — 自动评估，任务特定标准+标准化指标
Layer 4: Evolving       — 进化引擎（TextGrad/AFlow/MIPRO）
Layer 3: Workflow       — 工作流编排（从自然语言生成多 agent 流程）
Layer 2: Agent          — Agent 层（工具调用、memory、决策）
Layer 1: Basic          — 基础组件（LLM 接口、工具集、memory 模块）
```

## 三大进化算法

| 算法 | 作用 | 优化对象 |
|------|------|---------|
| **TextGrad** | 基于文本梯度优化 | Prompt 内容 |
| **MIPRO** | 结构与工作流优化 | 工作流拓扑 |
| **AFlow** | Prompt + 工作流联合进化 | 端到端 |

## 进化循环

```
定义目标（自然语言）
    ↓
自动生成工作流
    ↓
执行任务
    ↓
自动评估（内置评估器）
    ↓
进化优化（TextGrad/AFlow/MIPRO）
    ↓
更新 prompt/工具配置/工作流拓扑
    ↓
（循环）
```

## 评估指标体系

- **任务特定指标**：准确率、F1、代码通过率等
- **标准化基准**：HotPotQA（多跳推理）、MBPP（代码生成）、MATH（数学推理）、GAIA（通用AI助手）
- **内置评估器**：自动打分，不需要人工评判

## 其他关键特性

- **Human-in-the-Loop (HITL)**：支持人在回路——人审核、纠正、引导 agent 行为
- **Memory 系统**：短期（ephemeral）+ 长期（persistent），支持跨交互记忆
- **工具集**：代码执行、搜索引擎、数据库、文件系统、图像处理、浏览器

## 对 evo-skill-creator 的启示

### 直接可借鉴的

1. **自动评估器**：为每个智能体定义任务特定的评估标准
   - 操盘手：预判准确率、规则积分变化、持仓收益
   - 小说作家：读者反馈分数、章节完成度
   - 创建者自身：子智能体质量、知识库增长

2. **进化循环设计**：
   ```
   go(执行) → 自动评估 → 发现改进点 → 更新 skill/memory → 下次 go 更好
   ```
   这正是我们 backlog #12（自动化反馈循环）需要的

3. **自然语言定义目标**：用户描述"我要一个什么样的智能体"，系统自动生成工作流——我们的 go 命令方式 A 本质上就是这个

### 架构层面的参考

我们可以借鉴五层架构的分层思想：
- **评估层**（我们缺）→ review 命令可以扩展为自动评估
- **进化层**（我们缺）→ 目前只有手动 suggest/review，没有自动进化
- **工作流层**（我们有）→ go 命令的日常工作流程
- **Agent 层**（我们有）→ 每个智能体
- **基础层**（我们有）→ skill 文件、memory、工具

### 与 Memento-Skills 的互补

- Memento-Skills 聚焦"skill 存储和路由"
- EvoAgentX 聚焦"评估和进化算法"
- 两者结合 = 我们需要的完整进化体系
